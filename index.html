1. Introduction to JSON Web Tokens (JWT)

What is JWT?

JWT is a way to securely send information between parties as a JSON object. Think of it as a digital passport that proves who you are.

JWT Structure

HEADER
.
PAYLOAD
.
SIGNATURE

Example JWT:

eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VySWQiOiIxMjMiLCJuYW1lIjoiSm9obiJ9.abc123signature

JWT vs Sessions

  • Sessions: Server remembers who you are (like a hotel key card)
  • JWT: Token contains who you are (like carrying your ID everywhere)
  • JWT Benefits: No server storage needed, works across multiple servers
Simple Explanation: JWT is like a concert wristband that proves you paid for entry. Once you have it, you can enter any stage without asking permission again.

2. Generating and Verifying Tokens

Required Package

package.json
{ "name": "jwt-auth", "version": "1.0.0", "dependencies": { "express": "^4.18.0", "mongoose": "^7.0.0", "bcryptjs": "^2.4.3", "jsonwebtoken": "^9.0.0", "ejs": "^3.1.9" } }

JWT Basic Operations

const jwt = require('jsonwebtoken'); const SECRET_KEY = 'mysecretkey'; // Generate Token (Sign) const token = jwt.sign({ userId: user._id, name: user.name }, SECRET_KEY); // Verify Token const decoded = jwt.verify(token, SECRET_KEY); console.log(decoded); // { userId: "123", name: "John" }
Exam Tip: Only remember two methods:
  • jwt.sign() - Creates token
  • jwt.verify() - Checks token

Token Generation Process

  1. User logs in with email/password
  2. Server verifies credentials
  3. Server creates JWT with user info
  4. Client receives token and stores it
  5. Client sends token with every request

3. Protecting Routes with JWT

How Route Protection Works

Protected routes check if the user has a valid token before allowing access.

Simple Middleware

const authenticateToken = (req, res, next) => { const token = req.headers['authorization']; if (!token) { return res.status(401).send('Access denied. No token provided.'); } try { const decoded = jwt.verify(token, SECRET_KEY); req.user = decoded; next(); } catch (error) { res.status(400).send('Invalid token.'); } };

Using Middleware on Routes

// Public route (no protection needed) app.post('/login', (req, res) => { // Login logic here }); // Protected route (requires token) app.get('/dashboard', authenticateToken, (req, res) => { res.send(`Welcome ${req.user.name}!`); });
Flow:
  1. Client sends request with token in header
  2. Middleware checks if token exists
  3. Middleware verifies token with secret key
  4. If valid, request continues to route
  5. If invalid, return error message

4. Complete Single File Implementation

Complete JWT Authentication with EJS (One File)

jwt-app.js
const express = require('express'); const mongoose = require('mongoose'); const bcrypt = require('bcryptjs'); const jwt = require('jsonwebtoken'); const path = require('path'); const app = express(); const SECRET_KEY = 'mysecretkey'; // Connect to MongoDB mongoose.connect('mongodb://localhost:27017/jwtdb'); // User Schema const User = mongoose.model('User', { name: String, email: String, password: String }); // EJS Setup app.set('view engine', 'ejs'); app.set('views', path.join(__dirname, 'views')); // Middleware app.use(express.json()); app.use(express.urlencoded({ extended: true })); // JWT Middleware const authenticateToken = (req, res, next) => { const token = req.headers['authorization'] || req.cookies?.token; if (!token) { return res.status(401).json({ error: 'Access denied. No token provided.' }); } try { const decoded = jwt.verify(token, SECRET_KEY); req.user = decoded; next(); } catch (error) { res.status(400).json({ error: 'Invalid token.' }); } }; // Routes for Pages app.get('/', (req, res) => { res.render('login'); }); app.get('/register', (req, res) => { res.render('register'); }); app.get('/dashboard', authenticateToken, (req, res) => { res.render('dashboard', { user: req.user }); }); app.get('/profile', authenticateToken, (req, res) => { res.render('profile', { user: req.user }); }); // API Routes app.post('/api/register', async (req, res) => { const { name, email, password } = req.body; const hashedPassword = await bcrypt.hash(password, 10); const user = new User({ name: name, email: email, password: hashedPassword }); await user.save(); res.json({ message: 'User registered successfully!' }); }); app.post('/api/login', async (req, res) => { const { email, password } = req.body; const user = await User.findOne({ email: email }); if (!user) { return res.status(400).json({ error: 'User not found' }); } const isMatch = await bcrypt.compare(password, user.password); if (!isMatch) { return res.status(400).json({ error: 'Invalid password' }); } // Generate JWT Token const token = jwt.sign( { userId: user._id, name: user.name, email: user.email }, SECRET_KEY ); res.json({ message: 'Login successful', token: token, user: { name: user.name, email: user.email } }); }); app.listen(3000, () => { console.log('JWT Server with EJS running on port 3000'); });

Key Changes from HTML to EJS

  • Added EJS dependency in package.json
  • Set view engine: app.set('view engine', 'ejs')
  • Created separate EJS files in views/ directory
  • Added page routes: GET routes that render EJS templates
  • Dynamic content: User data passed from server to templates
  • Separated API routes: /api/ prefix for JSON responses

EJS Benefits:

  • Server-side rendering: Dynamic content generation
  • Template reusability: Include partials and layouts
  • Data binding: Pass server data directly to views
  • Better organization: Separate view files
  • SEO friendly: Content rendered on server
