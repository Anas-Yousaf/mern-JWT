1. Introduction to JSON Web Tokens (JWT)

What is JWT?

JWT is a way to securely send information between parties as a JSON object. Think of it as a digital passport that proves who you are.

JWT Structure

HEADER
.
PAYLOAD
.
SIGNATURE

Example JWT:

eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VySWQiOiIxMjMiLCJuYW1lIjoiSm9obiJ9.abc123signature

JWT vs Sessions

  • Sessions: Server remembers who you are (like a hotel key card)
  • JWT: Token contains who you are (like carrying your ID everywhere)
  • JWT Benefits: No server storage needed, works across multiple servers
Simple Explanation: JWT is like a concert wristband that proves you paid for entry. Once you have it, you can enter any stage without asking permission again.

2. Generating and Verifying Tokens

Required Package

package.json
{ "name": "jwt-auth", "version": "1.0.0", "dependencies": { "express": "^4.18.0", "mongoose": "^7.0.0", "bcryptjs": "^2.4.3", "jsonwebtoken": "^9.0.0" } }

JWT Basic Operations

const jwt = require('jsonwebtoken'); const SECRET_KEY = 'mysecretkey'; // Generate Token (Sign) const token = jwt.sign({ userId: user._id, name: user.name }, SECRET_KEY); // Verify Token const decoded = jwt.verify(token, SECRET_KEY); console.log(decoded); // { userId: "123", name: "John" }
Exam Tip: Only remember two methods:
  • jwt.sign() - Creates token
  • jwt.verify() - Checks token

Token Generation Process

  1. User logs in with email/password
  2. Server verifies credentials
  3. Server creates JWT with user info
  4. Client receives token and stores it
  5. Client sends token with every request

3. Protecting Routes with JWT

How Route Protection Works

Protected routes check if the user has a valid token before allowing access.

Simple Middleware

const authenticateToken = (req, res, next) => { const token = req.headers['authorization']; if (!token) { return res.status(401).send('Access denied. No token provided.'); } try { const decoded = jwt.verify(token, SECRET_KEY); req.user = decoded; next(); } catch (error) { res.status(400).send('Invalid token.'); } };

Using Middleware on Routes

// Public route (no protection needed) app.post('/login', (req, res) => { // Login logic here }); // Protected route (requires token) app.get('/dashboard', authenticateToken, (req, res) => { res.send(`Welcome ${req.user.name}!`); });
Flow:
  1. Client sends request with token in header
  2. Middleware checks if token exists
  3. Middleware verifies token with secret key
  4. If valid, request continues to route
  5. If invalid, return error message

4. Complete Single File Implementation

Complete JWT Authentication (One File)

jwt-app.js
const express = require('express'); const mongoose = require('mongoose'); const bcrypt = require('bcryptjs'); const jwt = require('jsonwebtoken'); const app = express(); const SECRET_KEY = 'mysecretkey'; // Connect to MongoDB mongoose.connect('mongodb://localhost:27017/jwtdb'); // User Schema const User = mongoose.model('User', { name: String, email: String, password: String }); // Middleware app.use(express.json()); // JWT Middleware const authenticateToken = (req, res, next) => { const token = req.headers['authorization']; if (!token) { return res.status(401).json({ error: 'Access denied. No token provided.' }); } try { const decoded = jwt.verify(token, SECRET_KEY); req.user = decoded; next(); } catch (error) { res.status(400).json({ error: 'Invalid token.' }); } }; // Register Route app.post('/register', async (req, res) => { const { name, email, password } = req.body; const hashedPassword = await bcrypt.hash(password, 10); const user = new User({ name: name, email: email, password: hashedPassword }); await user.save(); res.json({ message: 'User registered successfully!' }); }); // Login Route app.post('/login', async (req, res) => { const { email, password } = req.body; const user = await User.findOne({ email: email }); if (!user) { return res.status(400).json({ error: 'User not found' }); } const isMatch = await bcrypt.compare(password, user.password); if (!isMatch) { return res.status(400).json({ error: 'Invalid password' }); } // Generate JWT Token const token = jwt.sign( { userId: user._id, name: user.name, email: user.email }, SECRET_KEY ); res.json({ message: 'Login successful', token: token, user: { name: user.name, email: user.email } }); }); // Protected Route app.get('/dashboard', authenticateToken, (req, res) => { res.json({ message: `Welcome to dashboard, ${req.user.name}!`, user: req.user }); }); // Another Protected Route app.get('/profile', authenticateToken, (req, res) => { res.json({ message: 'This is your profile', userId: req.user.userId, name: req.user.name, email: req.user.email }); }); app.listen(3000, () => { console.log('JWT Server running on port 3000'); });

Exam Key Points

  • JWT = Token-based authentication (no server sessions)
  • jwt.sign(): Create token after login
  • jwt.verify(): Check token in middleware
  • Send token in headers: Authorization header
  • Protect routes: Use middleware before route handler

Quick Steps for Exam:

  3. Middleware: Check token with jwt.verify()
  4. Protected route: Add middleware before route
  5. Client: Send token in Authorization header

Minimal Version (20 lines)

minimal-jwt.js
const express = require('express'); const jwt = require('jsonwebtoken'); const app = express(); const SECRET = 'secret'; app.use(express.json()); // Login - generates token app.post('/login', (req, res) => { // Assume user is valid const token = jwt.sign({ userId: '123', name: 'John' }, SECRET); res.json({ token }); }); // Middleware const auth = (req, res, next) => { try { const token = req.headers.authorization; req.user = jwt.verify(token, SECRET); next(); } catch { res.status(401).json({ error: 'Invalid token' }); } }; // Protected route app.get('/dashboard', auth, (req, res) => { res.json({ message: `Hello ${req.user.name}` }); }); app.listen(3000);
